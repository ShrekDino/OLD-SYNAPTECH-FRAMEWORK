# Contributing to The Unified Blueprint

Thank you for your interest in this project. This is a living document — a work of theoretical reconstruction and constitutional architecture — and your engagement is valuable.

## Ways to Contribute

### 1. Feedback & Discussion
Open a [Discussion](https://github.com/QueerMarxist/The-Unified-Blueprint/discussions) for:
- General thoughts on the framework
- Theoretical questions or objections
- Historical examples you think should be addressed
- Suggestions for additional chapters or sections

### 2. Reporting Issues
Open an [Issue](https://github.com/QueerMarxist/The-Unified-Blueprint/issues) for:
- Factual errors or misattributions
- Typographical or formatting problems
- Broken references or citations
- Missing or unclear arguments

### 3. Proposing Changes
Submit a Pull Request if you'd like to propose:
- Corrections to existing text
- Additional supporting evidence or citations
- Refinements to the constitutional language in Appendix A
- New sections or chapters (please open a discussion first)

## Pull Request Guidelines

1. **Discuss first** — For non-trivial changes, open an issue or discussion before writing code
2. **One change per PR** — Keep focused on a single correction or addition
3. **Match the style** — Follow the existing formatting, citation style (Chicago footnotes), and tone
4. **Explain your reasoning** — In the PR description, explain what you changed and why

## Style Guide

- Written in **Markdown** (GFM compatible)
- Citations use **Chicago Manual of Style (full note)** format via `chicago-fullnote-bibliography.csl`
- Headings use ATX style (`#`, `##`, `###`)
- Footnotes use inline `^[citation]` syntax

## Code of Conduct

This project is committed to providing a welcoming and intellectually rigorous environment. All participants are expected to engage in good-faith discourse. Harassment, ad hominem attacks, or bad-faith arguments will not be tolerated.

## Neuro-Somatic Archive Contributions

Contributions to the companion [Neuro-Somatic Archive](neuro-somatic-archive/) follow the same standards.

### Phenotype Corrections
- Allostatic Load Profile inaccuracies
- Somatic Architecture Table errors (misattributed biological origins)
- Forensic Proxy Marker threshold corrections
- Entropy Threshold refinement
- Missing phenotype cross-references

### New Phenotypes
- Must include: Allostatic Load Profile, Somatic Architecture Table, Forensic Proxy Markers, Entropy Threshold (3-state), and Biological Bill of Rights
- Must map to an identified allostatic mechanism with quantifiable biological markers
- Zero-cost proxy protocols preferred over equipment-dependent assessments

### Proxy Protocol Submissions
- Must include: Physiological Substrate, Administration Protocol, Pathological Thresholds, Phenotype Cross-Reference
- Must be zero-cost and manually administrable (no equipment beyond timing device and penlight)
- Clinical validation data preferred

### Format
- Module files use `###` phenotype headings with subsections: **Allostatic Load Profile**, **Somatic Architecture Table**, **Forensic Proxy Markers**, **Entropy Threshold**, **Biological Bill of Rights**
- Tables are 3-column markdown with consistent headers
- CSV data files in `data/` must match the existing schema
