# Security Policy

## Supported Versions

The latest release and the `master` branch receive security updates. Older versions may receive critical patches on a best-effort basis.

| Version | Supported |
|---------|-----------|
| 0.1.x   | ✅ Active |
| < 0.1   | ❌ Not supported |

## Reporting a Vulnerability

If you discover a security vulnerability in DigiDollar, please report it privately. **Do not open a public GitHub issue.**

Send details to the repository maintainer through GitHub's private vulnerability reporting mechanism, or contact the project maintainer directly:

1. Go to the repository's **Security** tab
2. Click **Report a vulnerability**
3. Provide a detailed description, including steps to reproduce if possible

You can also reach out via the GitHub Issues page for a maintainer's contact information.

### What to include

- Type of vulnerability
- Steps to reproduce
- Impact description
- Suggested fix (optional but appreciated)

### Response timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 5 business days
- **Fix timeline**: Depends on severity; critical issues are prioritized

We appreciate responsible disclosure and will acknowledge your contribution in the release notes (unless you prefer to remain anonymous).
